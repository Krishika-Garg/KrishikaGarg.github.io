Web Penetration Testing Tool

Modern web applications often suffer from security flaws due to misconfigurations, while existing tools can be heavy or hard to integrate. This project is a lightweight, modular CLI tool designed to automate detection of common web vulnerabilities in a developer-friendly way.

Tools & Technologies
  • Language: Python
  • Core Concepts: Asynchronous programming, networking, CLI tooling
  • Security: TCP/UDP scanning, SSL/TLS analysis, authentication testing
  • Libraries: asyncio, socket, argparse, rich
  • Platform: Linux / Kali Linux
How It Works
  • Accepts a target domain or IP via CLI arguments with configurable scan options.
  • Uses asynchronous TCP and UDP scanning to probe multiple ports concurrently.
  • Performs modular security checks including authentication, SSL/TLS analysis, file and directory discovery, API endpoint scanning, and service exposure detection.
  • Analyzes HTTP responses, headers, cookies, and SSL certificates to identify weak configurations and information leakage.
  • Generates structured scan outputs for terminal review or export.
Results & Impact
  • Built a modular CLI security tool with 5+ core scanning modules, aligned with OWASP principles.
  • Improved scanning efficiency by ~40% using asynchronous execution.
  • Identified 10+ security vulnerabilities in controlled test environments.
  • Strengthened hands-on experience in networking, async system design, and real-world web security testing.

Federated Learning Under Data Poisoning Attacks

Federated Learning improves data privacy by keeping data on client devices, but it remains vulnerable to malicious participants. This project studies the impact of data poisoning and backdoor attacks on federated models and evaluates robust aggregation strategies to mitigate these threats.

Tools & Technologies
  • Language: Python
  • Core Concepts: Federated learning, distributed training, adversarial ML
  • Security & ML: Data poisoning, backdoor attacks, robust aggregation
  • Libraries: PyTorch, Flower (FLwr), Ray, NumPy, Pandas, Matplotlib
  • Platform: Google Colab
How It Works
  • Split the MNIST dataset across multiple simulated clients in an IID setup.
  • Each client trains a local CNN model on private data.
  • Malicious clients inject label-flip and backdoor attacks to bias learning.
  • Client updates are aggregated on a central server using multiple strategies.
  • Compared FedAvg, Trimmed Mean, Krum, and FoolsGold for robustness.
  • Evaluated global models using accuracy and attack success rate (ASR).
Results & Impact
  • FedAvg: 95.97% accuracy, 36.90% ASR (high vulnerability)
  • Trimmed Mean: 95.25% accuracy, 10.13% ASR (best trade-off)
  • Krum: 91.75% accuracy, 9.87% ASR (strong defense, lower accuracy)
  • FoolsGold: 95.85% accuracy, 28.78% ASR (partial protection)

Sales Data Analytics

Sales data is often large and fragmented, making it difficult to identify revenue drivers, performance trends, and customer behavior. This project analyzes sales data to uncover patterns, correlations, and actionable insights that support data-driven business and sales decisions.

Tools & Technologies
  • Language: Python
  • Core Concepts: Exploratory Data Analysis, trend analysis, correlation analysis
  • Data Analysis: Revenue trends, product and regional performance
  • Libraries: pandas, numpy, matplotlib, seaborn
  • Platform: Jupyter Notebook
How It Works
  • Processed and analyzed a sales dataset with 10,000+ transaction records across products, regions, and customer segments.
  • Cleaned data by handling <5% missing values, fixing inconsistencies, and standardizing formats.
  • Performed univariate analysis on sales, profit, and quantity to identify distributions and outliers.
  • Conducted time-series analysis over 12+ months to detect seasonality and long-term trends.
  • Compared revenue contribution across 5+ product categories and multiple regions.
  • Computed correlation metrics to understand relationships between sales revenue, profit, and order volume.
Results & Impact
  • Identified 2–3 peak sales periods, highlighting strong seasonal patterns.
  • Found that the top 20% of products generated ~60–65% of total revenue (Pareto-like distribution).
  • Observed moderate to strong correlations (0.45–0.65) between revenue, profit, and quantity sold.
  • Identified high-performing regions outperforming others by 30%+ in revenue.
  • Delivered insights supporting inventory planning, pricing strategy, and demand forecasting.

Banking Risk Data Analytics

Banks manage large volumes of customer data, yet identifying risk-prone customers and understanding patterns that influence financial risk remains challenging. This project analyzes banking customer data to uncover risk indicators and relationships that support better credit evaluation and risk assessment decisions.

Tools & Technologies
  • Language: Python
  • Core Concepts: Exploratory Data Analysis, risk analysis, feature understanding
  • Data Analysis: Customer profiling, correlation analysis, distribution analysis
  • Libraries: pandas, numpy, matplotlib, seaborn
  • Platform: Jupyter Notebook
How It Works
  • Cleaned and preprocessed banking data containing 5,000+ customer records, handling <4% missing values and correcting inconsistencies.
  • Performed univariate analysis on 10+ financial and demographic features to analyze distributions, skewness, and outliers.
  • Conducted multivariate analysis across multiple customer segments to uncover relationships between demographics and risk-related variables.
  • Applied correlation analysis, identifying moderate to strong relationships (≈0.4–0.6) between key financial attributes and risk indicators.
  • Created 15+ visualizations to clearly present trends, patterns, and actionable insights.
Results & Impact
  • Identified ~20–25% of customers as higher-risk segments based on financial and demographic patterns.
  • Observed moderate to strong correlations (0.4–0.6) between balance, income-related features, and risk indicators.
  • Found that a small subset of features explained most of the variance in customer risk behavior.
  • Demonstrated how exploratory analytics can enable early risk detection and more informed, data-driven decision-making.
